Gemini Platform Module

Executive Summary

A Terraform module for deploying Vertex AI Generative AI infrastructure on Google Cloud Platform, enabling Retrieval-Augmented Generation (RAG) and conversational AI with Gemini models. Provisions a Cloud Storage bucket for grounding data, a Discovery Engine Data Store for structured/unstructured content ingestion, a Discovery Engine Search Engine with enterprise-tier LLM add-ons, and an optional Dialogflow CX Agent for Agent Builder-powered conversational experiences. Supports configurable industry verticals, content configurations, and comprehensive labeling.

Component Breakdown

google_storage_bucket.grounding

Cloud Storage bucket for grounding data with configurable location, storage class, uniform bucket-level access, and versioning. Used to store documents, PDFs, and structured data that feeds into the Discovery Engine Data Store for RAG pipelines.

google_discovery_engine_data_store.this

Discovery Engine Data Store with configurable industry vertical (GENERIC, MEDIA, HEALTHCARE_FHIR) and content configuration (CONTENT_REQUIRED, NO_CONTENT, PUBLIC_WEBSITE). Indexes content for retrieval by the Search Engine. Solution type is fixed to SOLUTION_TYPE_SEARCH.

google_discovery_engine_search_engine.this

Enterprise-tier Search Engine with SEARCH_ADD_ON_LLM enabled for Gemini-powered search. Links to the Data Store and uses the default_collection. Configurable company name for personalized search results.

google_dialogflow_cx_agent.this

Optional Dialogflow CX agent for Agent Builder conversational AI. Features Stackdriver logging, spell correction, and speech-to-text adaptation. Configurable language code and time zone. Orchestrates search and Gemini model invocations for multi-turn conversations.

Data Flow

  • Data Ingestion: Documents, PDFs, and structured data are uploaded to the grounding GCS bucket.
  • Indexing: The Discovery Engine Data Store ingests and indexes content from the grounding bucket, creating searchable embeddings.
  • Search & Retrieval: The Search Engine (enterprise tier + LLM add-on) processes queries against the indexed content, returning relevant passages.
  • RAG Pipeline: Retrieved passages are combined with user queries as augmented context for the Gemini model, producing grounded responses.
  • Conversational AI: The Dialogflow CX agent orchestrates multi-turn conversations, invoking the Search Engine for retrieval and Gemini for response generation.
  • Speech Integration: The DFCX agent supports speech-to-text with adaptation for voice-based interactions.

Industry Adaptation

  • Enterprise Knowledge Base: Index internal documentation, policies, and FAQs in the Data Store. Gemini provides grounded answers via RAG. Dialogflow CX agent serves as an internal assistant.
  • Customer Support: Agent Builder-powered chatbot with enterprise search over support documentation, product manuals, and knowledge articles. Multi-language support via DFCX.
  • Healthcare: HEALTHCARE_FHIR industry vertical for medical knowledge bases. Grounded responses from clinical guidelines and research papers.
  • E-Commerce: Product catalog search with LLM-enhanced results. Conversational shopping assistant via Dialogflow CX with product recommendations.
  • Legal / Compliance: Index regulatory documents and contracts. RAG-powered search for clause identification and compliance queries.

Production Readiness Checklist

  • Enable versioning on the grounding bucket for data lineage
  • Configure uniform bucket-level access (default: true)
  • Set appropriate industry vertical for the Data Store (GENERIC, MEDIA, HEALTHCARE_FHIR)
  • Choose correct content config (CONTENT_REQUIRED for documents, PUBLIC_WEBSITE for web crawl)
  • Test Search Engine relevance before production deployment
  • Enable Dialogflow CX agent only when conversational AI is needed
  • Configure DFCX language code and time zone for target audience
  • Apply consistent labels for cost tracking across all resources
  • Set up data ingestion pipelines to keep the grounding bucket current
  • Monitor Search Engine quality metrics and tune as needed
  • Plan for data refresh cycles in the Data Store

Deployment

module "gemini_platform" {
  source = "github.com/kogunlowo123/terraform-gcp-gemini-platform"

  project_id = "my-project"
  region     = "us-central1"

  # Data Store
  data_store_id                = "prod-knowledge-base"
  data_store_display_name      = "Production Knowledge Base"
  data_store_industry_vertical = "GENERIC"
  data_store_content_config    = "CONTENT_REQUIRED"

  # Search Engine
  search_engine_id           = "prod-search"
  search_engine_display_name = "Production Search"

  # Grounding Bucket
  grounding_bucket_name     = "my-project-gemini-grounding-prod"
  grounding_bucket_location = "US"

  # Dialogflow CX Agent
  enable_dialogflow_agent       = true
  dialogflow_agent_display_name = "Production AI Assistant"
  dialogflow_agent_time_zone    = "America/New_York"

  labels = {
    environment = "production"
    team        = "ai-platform"
    cost_center = "ml-ops"
  }
}
